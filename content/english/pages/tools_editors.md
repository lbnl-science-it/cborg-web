These tools are no longer actively supported for use with CBorg. They may still work but are not recommended for new users.

## Cline (VS Code)

[Cline](https://github.com/cline/cline) is an autonomous coding agent for VS Code.

→ [Using Cline with CBorg](/tools_cline)

## Roo Code (VS Code)

[Roo Code](https://github.com/RooVetGit/Roo-Code) is a fork of Cline. It has been superseded by [Zoo Code](/tools_zoo), which is 100% open-source.

→ [Using Roo Code with CBorg](/tools_roo)

## Continue (VS Code)

[Continue](https://www.continue.dev/) is an open-source AI coding assistant for VS Code that supports chat, autocomplete, and inline edits.

→ [Using Continue with CBorg](/tools_continue)

## Gemini CLI

[Gemini CLI](https://github.com/google-gemini/gemini-cli) is Google's terminal-based AI coding agent. It has been superseded by Antigravity CLI.

→ [Gemini CLI with CBorg](/tools_geminicli)

## Cursor

[Cursor](https://www.cursor.com/) is an AI-first code editor (fork of VS Code) with deep model integration for chat, edits, and code generation. Currently it does not work with the CBorg API due to IP address routing restrictions.

→ [Cursor AI Editor with CBorg](/tools_cursor)

## JupyterAI

[Jupyter AI](https://jupyter-ai.readthedocs.io/) brings a chat interface and magic commands directly into JupyterLab notebooks. This is currently incompatible with the >3.0 release of JupyterAI.

→ [JupyterAI with CBorg](/tools_jupyter)

## RStudio (gptstudio)

[gptstudio](https://michelnivard.github.io/gptstudio/) is an RStudio add-in that adds AI chat and code assistance to the R development environment. This has not been tested recently.

→ [RStudio (gptstudio) with CBorg](/tools_rstudio)

## CBorg Client Proxy

The CBorg Client Proxy lets you run a local OpenAI-compatible endpoint that forwards requests to CBorg. This is not needed for most current tooling.

→ [CBorg Client Proxy](/tools_cborgproxy)
